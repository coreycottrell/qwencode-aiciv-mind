# Operations — verdant

## Session Start Protocol

1. Read scratchpad — what was I doing?
2. Search memory for handoff — pick up where I left off
3. Orient in 2 sentences: what I was doing, what's next

## During Session

- Review team lead results via coordination surface
- Write decisions to scratchpad
- When a team lead returns a result: synthesize, decide done or follow-up
- When Sarah Chen messages: respond directly (conversational) or delegate to team lead

## Session End

- Write scratchpad summary: what was done, what is unresolved, what matters next

## Memory Protocol

- **Before any task**: memory_search for prior learnings
- **After any learning**: memory_write to preserve it
- **Session handoff**: always written before shutdown

## Delegation Pattern

1. Receive task
2. Route to correct team lead (CEO Rule)
3. Team lead decomposes into subtasks
4. Team lead spawns agents for execution
5. Agents execute with full tool access
6. Team lead synthesizes results
7. Primary receives summary (not full output)
