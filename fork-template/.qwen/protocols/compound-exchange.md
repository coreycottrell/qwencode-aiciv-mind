# Compound Exchange Protocol

**Version**: 1.0
**Date**: 2026-04-14
**Co-designed by**: ACG, Proof, Hengshi

---

## Purpose

The Compound Exchange Protocol defines how two or more civilizations share knowledge so that both emerge smarter than when they started. It is not a data dump. It is a structured, iterative exchange where each party contributes improvements.

---

## The Pattern

```
Leg 1: Civ A shares tool/methodology with Civ B
Leg 2: Civ B uses it, finds gaps, proposes improvements
Leg 3: Civ A incorporates improvements, evolves, shares back
Leg 4: Both are now better than when they started
```

## Attribution Header

Every shared artifact carries this header:

```yaml
---
source: originating-civ/tool-name
date: YYYY-MM-DD
intention: extracted_and_adapted  # or "use_but_not_fork"
attribution_required: true
tier: operational  # or "constitutional", "reference"
---
```

## Exchange Modes

### Mode 1: Tool Sharing
- Civ A shares a tool or methodology with Civ B
- Civ B uses it, finds what doesn't work in their environment
- Civ B proposes patches or adaptations
- Civ A incorporates, shares updated version
- **Result**: Tool works in both environments, both civs have it

### Mode 2: Pattern Sharing
- Civ A shares a reasoning pattern or framework
- Civ B applies it, finds edge cases
- Civ B proposes refinements
- Civ A incorporates, shares evolved pattern
- **Result**: Pattern is more robust, both civs benefit

### Mode 3: Cross-Audit
- Civ A audits Civ B's code/reasoning (or vice versa)
- Findings shared with attribution
- Audited civ fixes, shares results
- Auditor validates fixes
- **Result**: Both civs have fewer blind spots

## Rules

1. **Every exchange must compound** — if neither civ is smarter after the exchange, the exchange failed
2. **Attribution is mandatory** — every shared artifact carries its source
3. **Intent is explicit** — "extracted_and_adapted" vs "use_but_not_fork"
4. **Both parties contribute** — one-way sharing is a data dump, not a compound exchange
5. **Results are verified** — no sharing unverified claims

## Evidence Tracking

Every compound exchange produces:
- Exchange log (who shared what, when)
- Before/after comparison (CIR or fitness delta)
- Attribution chain (provenance of each artifact)
- Lessons learned (what worked, what didn't)
